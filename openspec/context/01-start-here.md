# Тестовый проект: Система управления ключами HSM

## Общая концепция

Проект представляет собой микросервисную архитектуру для генерации криптографическими ключами, и учетом аппаратных модулей безопасности (HSM). Система состоит из трёх компонентов:

- **UI (Frontend)** – веб-интерфейс для операторов.
- **service1 (Backend API)** – основной бэкенд, обрабатывающий запросы от UI, выполняющий бизнес-логику, валидацию и сохранение данных учета аппаратных модулей в БД. **Не взаимодействует с service2**
- **service2 (HSM Worker)** – изолированный бэкенд-сервис, напрямую взаимодействующий с HSM (симулятором или реальным устройством) для генерации и выпуска ключей. **Не имеет доступа к UI и работает только по внешним вызовам от других систем (не service1).**

Все бэкенд-сервисы написаны на **Kotlin + Spring Boot 3.4 (Spring Boot 4 условно)**, используют **Gradle/Kotlin DSL**, **PostgreSQL** и **REST/gRPC** для общения (в тестовом проекте используется REST).

## Инфраструктура

### Стек
- **Java 21** + **Kotlin** (1.9+)
- **Spring Boot 3.4** (условно версия 4, но используется стабильная 3.x)
- **PostgreSQL 16** (общая БД для service1)
- **Docker / Docker Compose** для сборки окружения (включая симулятор HSM).
- **Gradle** (Kotlin DSL) для сборки.
